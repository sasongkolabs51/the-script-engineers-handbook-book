## 3. Reset Protocol Checklist

**Purpose:** A calm, structured response for when your chain produces degraded output and you can’t immediately tell why. Follow these steps; don’t panic.

- [ ] **Step 1 – Isolate the Layer.** Run each layer of your chain independently on a simple, well‑understood test topic (not the current script).  
  - Layer 1: Is the ideation beat sheet still coherent?  
  - Layer 2: Does the structural skeleton follow the framework when prompted alone?  
  - Layer 3: Does the pacing injection work on a clean, unvoiced skeleton?  
  - Layer 4: Does the voice overlay sound right on a neutral script?  
  - Identify exactly which layer is producing the degraded output.

- [ ] **Step 2 – Compare to a Known‑Good Snapshot.** Pull an example of strong output for that layer from your Engineering Log (or from a past script you loved). Compare side‑by‑side. What specific quality is missing? (e.g., “The hook is missing a concrete detail,” “The emotional temperature is flat,” “Pacing tokens are absent.”)

- [ ] **Step 3 – Adjust the Affected Layer Only.**  
  - If Layer 1: tighten the topic/angle/emotional‑outcome instructions.  
  - If Layer 2: revisit your beat script or guardrails; add a missing mandatory field.  
  - If Layer 3: refine your tempo map; provide more explicit token‑placement examples.  
  - If Layer 4: rerun the Voice Calibration Loop from Chapter 7 with fresh samples.  
  - Do NOT rebuild the whole system.

- [ ] **Step 4 – Document the Reset.** In your Failure Autopsy section, record:  
  - What triggered the degradation (model update, new format, fatigue, etc.)?  
  - What fixed it?  
  - What can you do proactively to prevent it?  
  (e.g., “After model update, pacing tokens are being ignored in longer scripts. Fix: added an explicit token‑preservation rule to Layer 3 prompt. Prevention: will run a short pacing test with each major model update.”)

**Emotional guard:** When output degrades, remind yourself: “I’m not losing my touch. A layer needs recalibration. I can find and fix it.”
