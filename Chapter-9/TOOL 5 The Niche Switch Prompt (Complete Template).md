## TOOL 5: The Niche Switch Prompt (Complete Template)

### *The integrated prompt that retunes your entire system for a target niche in one pass. The synthesis of every pillar.*

---

```
═══ NICHE SWITCH PROMPT ═══

I have a script kernel that I need to adapt for a specific niche.
Apply the full niche configuration below, modifying structure,
hook, voice, and pacing simultaneously.

═══ SCRIPT KERNEL (niche-independent) ═══

TOPIC: [your universal topic]
CORE POINTS:
- [Point 1]
- [Point 2]
- [Point 3]
- [Point 4]
KEY INSIGHT / TAKEAWAY: [the universal lesson]
PERSONAL ELEMENT: [your story/experience/data that anchors this]

═══ TARGET NICHE CONFIGURATION ═══

NICHE: [Educational / Reaction / Story-Vlog / Review / Other: ___]
AUDIENCE: [specific viewer description for this niche]
AUDIENCE EXPECTATION: [what they expect from this content type —
e.g., "clarity and skill transfer" / "emotional connection and
catharsis" / "authority and a clear verdict"]

FRAMEWORK: [your chosen framework for this niche]
BEAT SCRIPT: [paste the relevant beat script from Chapter 6,
or describe the beat sequence]

HOOK:
- Archetype: [from Chapter 5]
- Retention mechanism: [Open Loop Extension / Pattern Interrupt /
  Stake Escalation]
- Tempo: [from Tool 6 hook-tempo mapping]
- Specific instruction: [e.g., "Open with a Specific Promise:
  state that the viewer will leave with [specific capability].
  Anchor with [specific proof]."]

VOICE MODIFICATIONS (applied ON TOP of my baseline Voice Profile):
[PASTE YOUR VOICE MODIFICATION BLOCK FROM TOOL 3]

PACING SIGNATURE:
[PASTE YOUR PACING MODIFICATION BLOCK FROM TOOL 4]

NICHE-SPECIFIC GUARDRAILS:
1. [e.g., "Never introduce a concept without a concrete example."]
2. [e.g., "Every claim must be backed by a specific test or number."]
3. [e.g., "Never rush an emotional beat."]

CROSS-NICHE SAFEGUARDS (NEVER change these):
[PASTE FROM TOOL 6]

═══ OUTPUT ═══

Generate a complete script skeleton (Layer 2 level) that:
1. Follows the specified framework beat sequence
2. Opens with the specified hook archetype + retention mechanism
3. Applies the voice modifications throughout
4. Includes pacing tokens per the niche pacing signature
5. Includes B-roll cues per the niche B-roll style
6. Respects all guardrails and safeguards
7. Preserves the core insight from the kernel

After the script, add a "Niche Fidelity Check":
- Hook signals the correct niche? [Y/N]
- Structure matches audience expectations? [Y/N]
- Voice is recognizably mine but in the right register? [Y/N]
- Pacing feels native to this content type? [Y/N]
- Core insight preserved? [Y/N]
```
