## 9. Troubleshooting Quick Reference

| **Problem** | **Likely Cause** | **Fix** |
|-------------|------------------|---------|
| Script reads like a summary despite using a framework | The AI defaulted to informational mode; guardrails missing or too weak | Add the Summary Prohibition Guard and the Concreteness Guard. Include an example of the kind of language you want. |
| Emotional arc feels flat | The framework name was used as a label, not a beat script | Decompose the framework into explicit beats with emotional job descriptions. Prompt each beat individually if necessary. |
| Beat boundaries are fuzzy; two beats merged | Prompt length too long; AI lost track | Split the framework across multiple prompts (e.g., Beats 1–4 in one, Beats 5–7 in a follow‑up). |
| Final beat doesn’t reference the opening | The Payoff‑Reference Guard wasn’t active | Add: “The final beat must include a specific callback—a phrase, a setting, or a sensory detail—that appeared in Beat 1. This loop closure is mandatory.” |
| Stakes don’t escalate | The beat descriptions didn’t require escalation | Add the Stake‑Escalation Guard and specify the emotional threshold for each beat. |
