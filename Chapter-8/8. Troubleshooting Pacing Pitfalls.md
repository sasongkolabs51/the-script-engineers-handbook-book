## 8. Troubleshooting Pacing Pitfalls

| Symptom | Likely Cause | Fix |
|---------|--------------|-----|
| Tokens feel randomly placed; rhythm is jarring. | No tempo map was used; AI guessed. | Build a rhythm grid for this framework and paste the tempo map into the prompt. |
| Script feels too slow and heavy with pauses. | Overuse of pause tokens. | Reduce pause frequency; use [Slow down] instead of multiple pauses. Reserve [Pause 3s] for only the most dramatic moments. |
| Script feels rushed; no breathing room. | Not enough pauses or speed‑shift tokens. | Increase pause tokens after key statements. Ensure at least one [Pause 1.5s] after each major beat transition. |
| AI ignores tokens in output. | Tokens not distinct or prompt didn’t demand preservation. | Format tokens in bold brackets `[Pause 1s]`. Add: “You must include the tokens exactly as specified, inline.” |
| Voice overlay (Layer 4) deletes or rewrites tokens. | Layer 4 prompt doesn’t protect tokens. | Add to Voice Overlay prompt: “Keep all pacing tokens and their placement unchanged. Only revoice the surrounding text.” |
| B‑roll moments are generic (“show a laptop”). | B‑Roll prompt not specific enough. | Add: “Describe the exact scene, object, or interface with detail—brand, color, action, comparison.” |
| Rhythm feels unnatural when read aloud. | Tokens conflict with natural sentence rhythm. | Read aloud and adjust token placement manually in the final script; then update the tempo map for next time. |
