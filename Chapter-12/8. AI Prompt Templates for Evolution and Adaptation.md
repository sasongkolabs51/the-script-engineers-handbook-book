## 8. AI Prompt Templates for Evolution and Adaptation

### 8.1 Template Evolution Loop Assistant

```
I’m running a Template Evolution Loop on one of my script templates. Here’s the data:

- Template name and current version: [name, version]
- Video topic: [topic]
- Key metrics this week: [e.g., retention at 30s = 62%, subscribe rate = 2.1%]
- Retention graph observation: [describe where the curve dropped]
- My hypothesis about which beat/layer failed: [e.g., "The transition from hook to first agitation beat lost viewers"]

Based on this, suggest:
1. One specific adjustment to a beat’s emotional job, mandatory field, or guardrail that could address the drop.
2. How to phrase that adjustment as a prompt instruction.
3. A way to test this change in my next video without introducing other variables.
```

### 8.2 Platform Adaptation Prompt

```
I’m adapting a script kernel for a new platform. Here’s what I have:

- Universal script kernel (topic and core points): [paste]
- My baseline Voice Profile: [paste summary]
- Target platform: [TikTok / Reels / LinkedIn Video / Live Stream / Other]
- My goal for this adaptation: [e.g., create a vertical short-form hook that stops the scroll in under 1.5 seconds]

Please propose a complete platform‑specific prompt modification that adjusts hook archetype, pacing signature, voice settings, and visual token density for this platform. Base it on typical platform grammar (vertical, fast, visual) and my voice profile. Output the adapted script opening and a summary of the changes made.
```

### 8.3 New Framework Brainstormer

```
I want to invent a new storytelling framework for a specific type of video. Here’s the genre: [describe, e.g., "a video essay that blends personal confession with historical analysis"].

Please suggest a possible beat structure (5‑7 beats), with an emotional job and mandatory field for each beat. Then suggest two guardrails that would prevent the AI from defaulting to a dry, academic tone. Output the structure in a format I can turn into a reusable prompt template.
```
